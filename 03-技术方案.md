# 技术方案

## 技术栈

复用 AI Detector 项目已验证的框架，降低开发风险。

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| 框架 | Next.js 14 | App Router，支持 SSG/SSR |
| 语言 | TypeScript | 类型安全 |
| 样式 | Tailwind CSS | 快速布局 |
| 部署 | Cloudflare Pages | 免费套餐，全球 CDN |
| 域名 | Cloudflare Registrar | counteruavhub.com |
| 分析 | Google Analytics 4 | 流量追踪 |
| 搜索收录 | Google Search Console | SEO 监控 |

## 数据方案

MVP 阶段频率数据直接写在 `data/drones.ts` 静态文件中，无需数据库。

数据结构：
```typescript
interface DroneRecord {
  id: string
  name: string
  brand: string
  category: 'consumer' | 'industrial' | 'fpv' | 'military'
  controlFreq: string      // e.g. "2.4GHz / 5.8GHz"
  videoFreq: string        // e.g. "5.8GHz"
  gpsFreq: string          // e.g. "1.575GHz (L1)"
  counterFreq: string      // 建议反制频率
  source: string           // 数据来源
}
```

## 内容方案

博客文章使用 Markdown 文件存储在 `content/blog/` 目录，通过 Next.js 静态生成页面。

## 部署方案

- Git push 到 GitHub 主分支 → Cloudflare Pages 自动部署
- 构建命令：`next build`
- 输出目录：`.next`

## SEO 技术配置

- `next/metadata` 配置每页 title / description
- `next-sitemap` 自动生成 sitemap.xml
- JSON-LD schema markup（工具页用 WebApplication，文章页用 Article）
- OpenGraph 图片（静态生成）
